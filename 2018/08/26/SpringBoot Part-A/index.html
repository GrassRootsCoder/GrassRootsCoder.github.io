Using Spring Boot

本章的主题包括构建系统和如何启动你的应用。Spring Boot本身没有什么特别的东西，不过按照本章节的内容来构建项目会事半功倍。

这部分内容可以参考代码 springboot-structure 项目。

Build Project

创建 Maven 工程，并添加 POM 文件依赖

父工程 POM 依赖

子工程 springboot-structure POM 依赖

在父工程依赖中，添加 SpringBoot 标签，添加这个标签后，去设置 spring-boot-starter-* 依赖时可以忽略版本属性，同时方便统一管理版本，推荐使用这种方式。

在子工程依赖中，因为已经存在 spring-boot-starter-parent 依赖，后续添加依赖不再需要版本属性。SpringBoot 提供了很多 starter 依赖，可以参考这个文档中内容（链接）。

项目包结构

不同公司对于项目包结构有不同的规则，下面是我目前公司的规则，值得注意的是：主应用程序类（添加@SpringBootApplication 注解）需要放在包的根路径下，会自动加载包路径下所有添加了相关注解（@Service、@Controller…这些）的类。否则需要添加额外的属性来加载不在包路径下的类。

@SpringBootApplication 注解中存在 scanBasePackages 和 scanBasePackageClasses 这两个字段，第一个是用来加载包路径（指定包路径），第二个是直接加载类（指定类全限定名称）。不管哪种方式都比较繁琐，因此推荐将主应用程序类放在包的根路径下。

配置类和文件

在 SpringBoot 工程中会使用到配置文件和配置类，配置文件一般放在 resources 目录下，格式可以是 properties 或者 yml；配置类是 Java 代码编写的，替代原有 XML 方式声明类。

Run Project

IDEA 启动

使用 IDEA 直接运行主程序类，在控制台可以看到程序运行成功。

Jar 启动

执行 Maven 打包命令，在命令行中直接运行打包好的 Jar 包。

Spring Boot Feature

接下来介绍一些 SpringBoot 常用功能和特性，一般我们搭建 SpringBoot 工程都是用来做 Web 服务器，主要包含以下三个基本点：

  1. 和前端进行交互。处理请求和返回数据。
  2. 全局处理异常。
  3. 校验是否重复提交（涉及注解）。
  4. 和数据库进行交互。
  5. 打印业务日志，用以观察线上运行情况。
