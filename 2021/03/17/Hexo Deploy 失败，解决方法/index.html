Hexo Deploy 失败，解决方法

Posted on 2021-03-17 | Post modified: 2021-03-17 | In Hexo Question

碰到 hexo deploy 命令执行失败情况很多次，记录一下如何解决的。


Hexo deploy 失败情况1

出现 LibreSSL SSL_connect: SSL_ERROR_SYSCALL in connection to github.com:443 这个问题表示链接不上 Github，一般来说是开了代理或者是域名解析不对。

解决办法有三种

1.在 git 中添加代理配置

“” 中的内容替换成自己的代理地址 “proxy.domain:port”，具体地址需要在全局代理配置里面找到


2.取消 git 代理


3.在 hosts 中添加 github.com 配置

在 /etc/hosts 中添加 13.229.188.59 github.com 配置


我这次出现问题的原因是 github 域名解析，所以用得是第三种解决方法，如果是开了全局代理的，可以使用前面两种办法。有时候第三种办法同样也会出现问题，这个是网络波动造成的（太卡了）。


Hexo deploy 失败情况2

出现 Error: EACCES: permission denied, unlink 这个情况是 .deploy_git 文件夹有问题。

产生原因初步估计：原来我 Hexo git 仓库配置是 HTTPS 方式，后来修改成 SSH 方式，接着又修改回 HTTPS 方式，就有这个问题了。（为啥改来改去？在尝试解决之前的一个问题。。）

解决办法，删除 .deploy_git 文件夹，重新执行 hexo d 命令
